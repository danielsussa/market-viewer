Help: how to generate a Market Viewer URL

This page explains the complete format for the ?payload=... query parameter. It is designed for automation use-cases, including AI bots/agents that generate reports and market snapshots.

1) URL format

https://YOUR_DOMAIN/?payload=ENCODED_PAYLOAD

GitHub Pages DNS for this project: https://danielsussa.github.io/market-viewer/

The app accepts payload as raw JSON, URL encoded, base64, or base64url. Recommended: base64url (safer for query strings).

2) Minimum JSON structure

{
  "title": "WINJ26",
  "candles": [
    { "time": "2026-03-05T13:10:00Z", "open": 131150, "high": 131220, "low": 131120, "close": 131205 },
    { "time": "2026-03-05T13:11:00Z", "open": 131205, "high": 131240, "low": 131180, "close": 131210 }
  ]
}
  • title is optional (fallback: symbol or “Market Viewer”).
  • candles or candes are both supported.
  • OHLC can be open/high/low/close or o/h/l/c.
  • time is optional: ISO, unix sec, or unix ms. If missing, minute-by-minute sequence is generated.

2.1) Auto export via query params (PNG/JPG)

You can export the current chart image automatically on load (no UI button needed):

  • export=png or export=jpg (also accepts jpeg)
  • download=1|0 (default: 1)
  • filename=your-name (optional, extension is added automatically)
  • quality=0..1 (optional, JPG only; default 0.92)
  • exportDelay=ms (optional, delay before capture; default 120)
https://YOUR_DOMAIN/?payload=ENCODED_PAYLOAD&export=png&download=1&filename=snapshot

The export snapshot includes chart canvas + overlays (objects) + header (title/subtitle), matching the current theme rendered on screen.

3) Overlay objects (objects)

Supported types:

  • vertical-line (vertical line at a given time)
  • horizontal-line (horizontal line at a given price)
  • text (label at time+price or absolute x+y)
  • buy-arrow / sell-arrow (native chart markers)
{
  "objects": [
    { "type": "vertical-line", "time": "2026-03-05T13:10:00Z", "label": "NY Open", "color": "#ffcc00" },
    { "type": "horizontal-line", "price": 131200, "label": "Level", "color": "#78c8ff" },
    { "type": "text", "text": "Attention zone", "time": "2026-03-05T13:18:00Z", "price": 131205 },
    { "type": "buy-arrow", "time": "2026-03-05T13:18:00Z", "label": "Buy" },
    { "type": "sell-arrow", "time": "2026-03-05T13:24:00Z", "label": "Sell" }
  ]
}

4) Horizontal and vertical grid (optional)

You can enable/disable grid lines in payload:

{
  "grid": {
    "vertical": true,
    "horizontal": false
  }
}

Also supported aliases:

  • grid.vertLines and grid.horzLines
  • gridVertical and gridHorizontal at root level

Accepted values: boolean, 1/0, true/false, yes/no, on/off.

You can also control opacity and line style:

{
  "grid": {
    "vertical": true,
    "horizontal": true,
    "opacity": 0.2,
    "verticalOpacity": 0.35,
    "horizontalOpacity": 0.15,
    "style": "dashed",
    "verticalStyle": "dotted",
    "horizontalStyle": "solid"
  }
}

Style values: solid, dotted, dashed, large-dashed, sparse-dotted.

5) AI bot generation recipe

  1. Generate valid JSON with a candles array.
  2. Validate OHLC per candle: low ≤ min(open,close) and high ≥ max(open,close).
  3. Sort candles by ascending time.
  4. Serialize JSON compactly.
  5. Encode as base64url.
  6. Build final URL: BASE_URL + "?payload=" + encoded.

Tip: avoid very large query strings. If browser/proxy limits are hit, reduce candle count.

6) JavaScript example (Node/browser)

const payload = {
  title: "WINJ26",
  candles: [
    { time: "2026-03-05T13:10:00Z", open: 131150, high: 131220, low: 131120, close: 131205 },
    { time: "2026-03-05T13:11:00Z", open: 131205, high: 131240, low: 131180, close: 131210 }
  ],
  grid: { vertical: true, horizontal: true },
  objects: [
    { type: "buy-arrow", time: "2026-03-05T13:11:00Z", label: "Entry" }
  ]
}

const json = JSON.stringify(payload)
const b64url = btoa(json).replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/g, '')
const url = `http://localhost:5173/?payload=${b64url}`
console.log(url)

7) Python example

import json, base64

payload = {
  "title": "WINJ26",
  "candles": [
    {"time": "2026-03-05T13:10:00Z", "open": 131150, "high": 131220, "low": 131120, "close": 131205},
    {"time": "2026-03-05T13:11:00Z", "open": 131205, "high": 131240, "low": 131180, "close": 131210}
  ],
  "grid": {"vertical": True, "horizontal": False}
}

raw = json.dumps(payload, separators=(",", ":"), ensure_ascii=False).encode("utf-8")
b64url = base64.urlsafe_b64encode(raw).decode("utf-8").rstrip("=")
url = f"http://localhost:5173/?payload={b64url}"
print(url)

8) Shell example (jq + base64)

PAYLOAD=$(jq -nc '{
  title:"WINJ26",
  candles:[
    {time:"2026-03-05T13:10:00Z",open:131150,high:131220,low:131120,close:131205},
    {time:"2026-03-05T13:11:00Z",open:131205,high:131240,low:131180,close:131210}
  ],
  grid:{vertical:true,horizontal:false}
}' | base64 | tr -d '\n' | tr '+/' '-_' | tr -d '=')

open "http://localhost:5173/?payload=${PAYLOAD}"

9) Common errors

  • “query param payload is required”: missing ?payload=....
  • “invalid payload”: broken encoding (invalid base64 / URL encoding).
  • “payload has no candles/candes array”: JSON has no candle array.
  • “invalid candle[n]”: non-numeric or inconsistent OHLC/time.

If you automate with AI, always validate JSON before generating the final URL.
