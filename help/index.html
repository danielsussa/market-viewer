Help: como gerar a URL do Market Viewer

Esta página explica o formato completo para montar o parâmetro ?payload=.... O objetivo é facilitar consumo por bots/agents de IA que geram relatórios e snapshots de mercado.

1) Formato da URL

https://SEU_DOMINIO/?payload=ENCODED_PAYLOAD

O app aceita payload em JSON raw, URL encoded, base64 ou base64url. Recomendado: base64url (mais robusto para query string).

2) Estrutura mínima do JSON

{
  "title": "WINJ26",
  "candles": [
    { "time": "2026-03-05T13:10:00Z", "open": 131150, "high": 131220, "low": 131120, "close": 131205 },
    { "time": "2026-03-05T13:11:00Z", "open": 131205, "high": 131240, "low": 131180, "close": 131210 }
  ]
}
  • title opcional (fallback: symbol ou “Market Viewer”).
  • candles ou candes (ambos aceitos).
  • OHLC pode ser open/high/low/close ou o/h/l/c.
  • time opcional: ISO, unix sec ou unix ms. Sem time, usa sequência por minuto.

3) Objetos extras (objects)

Tipos suportados:

  • vertical-line (linha vertical em um tempo)
  • horizontal-line (linha horizontal em um preço)
  • text (rótulo em time+price ou em x+y)
  • buy-arrow / sell-arrow (setas nativas do chart)
{
  "objects": [
    { "type": "vertical-line", "time": "2026-03-05T13:10:00Z", "label": "Abertura NY", "color": "#ffcc00" },
    { "type": "horizontal-line", "price": 131200, "label": "Nível", "color": "#78c8ff" },
    { "type": "text", "text": "Zona de atenção", "time": "2026-03-05T13:18:00Z", "price": 131205 },
    { "type": "buy-arrow", "time": "2026-03-05T13:18:00Z", "label": "Compra" },
    { "type": "sell-arrow", "time": "2026-03-05T13:24:00Z", "label": "Venda" }
  ]
}

4) Grid horizontal e vertical (opcional)

Você pode ligar/desligar o grid diretamente no payload:

{
  "grid": {
    "vertical": true,
    "horizontal": false
  }
}

Aliases também aceitos:

  • grid.vertLines e grid.horzLines
  • gridVertical e gridHorizontal na raiz

Valores aceitos: boolean, 1/0, true/false, yes/no, on/off, sim/não.

5) Receita para bots de IA (passo a passo)

  1. Gerar JSON válido com candles.
  2. Validar OHLC por candle: low ≤ min(open,close) e high ≥ max(open,close).
  3. Ordenar candles por tempo crescente.
  4. Serializar JSON sem quebras desnecessárias.
  5. Codificar em base64url.
  6. Montar URL final: BASE_URL + "?payload=" + encoded.

Dica: evite payload gigantesco em query string. Se passar do limite do navegador/proxy, reduza candles.

6) Exemplo em JavaScript (Node/browser)

const payload = {
  title: "WINJ26",
  candles: [
    { time: "2026-03-05T13:10:00Z", open: 131150, high: 131220, low: 131120, close: 131205 },
    { time: "2026-03-05T13:11:00Z", open: 131205, high: 131240, low: 131180, close: 131210 }
  ],
  grid: { vertical: true, horizontal: true },
  objects: [
    { type: "buy-arrow", time: "2026-03-05T13:11:00Z", label: "Entrada" }
  ]
}

const json = JSON.stringify(payload)
const b64url = btoa(json).replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/g, '')
const url = `http://localhost:5173/?payload=${b64url}`
console.log(url)

7) Exemplo em Python

import json, base64

payload = {
  "title": "WINJ26",
  "candles": [
    {"time": "2026-03-05T13:10:00Z", "open": 131150, "high": 131220, "low": 131120, "close": 131205},
    {"time": "2026-03-05T13:11:00Z", "open": 131205, "high": 131240, "low": 131180, "close": 131210}
  ],
  "grid": {"vertical": True, "horizontal": False}
}

raw = json.dumps(payload, separators=(",", ":"), ensure_ascii=False).encode("utf-8")
b64url = base64.urlsafe_b64encode(raw).decode("utf-8").rstrip("=")
url = f"http://localhost:5173/?payload={b64url}"
print(url)

8) Exemplo em shell (jq + base64)

PAYLOAD=$(jq -nc '{
  title:"WINJ26",
  candles:[
    {time:"2026-03-05T13:10:00Z",open:131150,high:131220,low:131120,close:131205},
    {time:"2026-03-05T13:11:00Z",open:131205,high:131240,low:131180,close:131210}
  ],
  grid:{vertical:true,horizontal:false}
}' | base64 | tr -d '\n' | tr '+/' '-_' | tr -d '=')

open "http://localhost:5173/?payload=${PAYLOAD}"

9) Erros comuns

  • “query param payload é obrigatório”: faltou ?payload=....
  • “payload inválido”: codificação quebrada (base64/url-encoding inválido).
  • “payload sem candles/candes”: JSON não possui array de candles.
  • “candle[n] inválido”: OHLC/time não numéricos ou inconsistentes.

Se você está automatizando com IA, sempre valide o JSON antes de montar a URL.
